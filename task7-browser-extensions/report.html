Summary

The audit identifies extensions with broad permissions or suspicious behavior. This package contains steps to inspect, flags to look for, and remediation actions. Follow the instructions to reproduce and document your findings.

Audit Steps

Open chrome://extensions/, enable Developer mode, and inspect each extension details. Review permissions, host access, background scripts, and store page.

Command Snippets

# PowerShell: list Chrome extensions folder names and manifest names
$extDir = "$env:LOCALAPPDATA\Google\Chrome\User Data\Default\Extensions"
Get-ChildItem $extDir -Directory | ForEach-Object {
  $id = $_.Name
  $latest = Get-ChildItem $_.FullName -Directory | Sort-Object Name -Descending | Select-Object -First 1
  $manifest = Join-Path $latest.FullName 'manifest.json'
  $name = if (Test-Path $manifest) { (Get-Content $manifest -Raw | ConvertFrom-Json).name } else { 'Unknown' }
  Write-Output "$id - $name - $latest"
}
